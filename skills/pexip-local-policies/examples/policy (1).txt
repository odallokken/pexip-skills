Add textoverlay to gateway rules (primarely for Skype Meetings)

This local policy service config will check if a call is a gateway call, and if the call routing rule we are hitting has "overlaytext" as the start of the service tag.

{
     
    {# Checking if there is a valid service config for the dialed alias #}
 
    {% if service_config  %}
        "action" : "continue",
 
        {# Checking if service_type = gaeway, AND the tag starts with "overlaytext" #}
 
        {% if service_config.service_type == "gateway" and service_config.service_tag.startswith("overlaytext") %}
 
 
            {# if it does, flip the switch for enable_overlay_text to true #}
 
            "result" : {{service_config|pex_update({"enable_overlay_text": true})|pex_to_json}}
 
        {% else %}
 
            {# if it does not - no changes done #}
 
            "result" : {{service_config|pex_to_json}}
 
        {% endif %} 
 
    {# If there were no service config for the dialed alias, the call will be rejected (as it normally would) #}
 
    {% else %}
        "action" : "reject",
        "result" : {}
    {% endif %}
}

Policy #3


Adding an ADP (Automated dialed participant)


To add an ADP via local policy, this can't be done with pex_update since that will overwrite what's in the automatic_participants array.

Instead, you will need to do this:

{% set adps = service_config.automatic_participants %}
{% if adps %}
  {{ adps.extend( [{
    "description": "Extra ADP", 
    "local_alias": "extraadps@example.com", 
    "local_display_name": "Call from the VMR", 
    "protocol": "sip", 
    "remote_alias": "sip:extraadp@example.com", 
    "remote_display_name": "Miss Annable De Participant", 
    "role": "chair", 
    "system_location_name": "London"
    }]) or "" }}
{% endif %}

{
  {% if service_config %}
    "action" : "continue",
    "result" : {{service_config|pex_to_json}}
  {% else %}
    "action" : "reject",
    "result" : {}
  {% endif %}
}
This obviously can be put inside an "if" block. Note two things: the 'or "" ' since we need a string, not None, in this situation, and note also that this updates service_config directly so no need to do any pex_updating.

Policy #5

Auto create VMR and ADP to Skype Meeting


This policy will take any call with ;gruu;opaque=app:conf:focus:id: and in stead of doing separate GW call (meaning VC endpoints communicate via the Skype meeting) it will create a ad-hoc VMR, and auto dial out to the SfB meeting.

Advantages: Better audio/video quality between VC endpoints, lower HW usage (with GW: 3.5 HD, with VMR: 1 HD), video endpoints can still see a mixed layout of SfB participants and Video endpoints.
Disadvantages: SfB users does not get roster list control, SfB users see one merged participant stream being the ad-hoc VMR, default 1+7 layout in this script, SfB users with a conference where they have to admit every participant, will only have control of the first one, the next one is admitted automatically given that the SfB meeting has admitted the ad-hoc VMR, name in the participant list in SfB will be a generic name (name of the ADP dialout)
Review any parts of this script that has a comment with pexCheck to decide how you would like the script to run in your deployment.

{
    {# Remove outer hashes from the two next line to print debug info in support log - recomment before going prod! #}
    {#{pex_debug_log("##### call_info ##### ", call_info) }#}
    {#{pex_debug_log("##### service_config ##### ", service_config) }#}
 
    {% set s4buser = call_info.local_alias|pex_regex_replace('^sip:(.+)@(.+);gruu;opaque=app:conf:focus:id:([a-z0-9A-Z]+)', '\\1' ) %}
    {% set s4bdomain = call_info.local_alias|pex_regex_replace('^sip:(.+)@(.+);gruu;opaque=app:conf:focus:id:([a-z0-9A-Z]+)', '\\2') %}
    {% set s4bgruu   = call_info.local_alias|pex_regex_replace('^sip:(.+)@(.+);gruu;opaque=app:conf:focus:id:([a-z0-9A-Z]+)', '\\3') %}
 
 
         {# Checking if call_info.local_alias contains a GRUU #}
         {% if  ";gruu;opaque=app:conf:focus:id:" in call_info.local_alias %}
            {{pex_debug_log("-----------------  MATCH for GRUU OPAQUE = APP CONF FOCUS ID ----------------- ") }}
            {{pex_debug_log("service_config Policy : Adding ADP for SFB conference ", call_info.location, " protocol", call_info.protocol, " destination user", s4buser, " destination domain", s4bdomain, " destination GRUU", s4bgruu, " FROM: ", call_info.remote_alias) }}
 
            "action" : "continue",
            "result" : {
                "service_type" : "conference",
                "name" : "{{s4buser}}@{{s4bdomain}} Skype Meeting ({{s4bgruu}})",   {# pexCheck: Name format of VMR #}
                "service_tag" : "Triggered by first Video endpoint: {{call_info.remote_alias}} ({{call_info.protocol}}) - S4B USER {{s4buser}} - GRUU {{s4bgruu}}",
                "description" : "{{s4buser}} ({{s4bgruu}}) - trigger {{call_info.remote_alias}} ({{call_info.protocol}})",
 
                 {# pexCheck: Set a random long PIN and allow_guests = true if you prefer VC endpoints to not join the merged VMR prior  #}
                 {#  to first SfB user letting them in, VC endpoints will also be teared down after configured teardown buffer in global config #}
                "pin" : "",
                "allow_guests" : false,
                "guest_pin" : "",
 
                {# pexCheck: Uncomment (remove the first/last two characters of the line) the next line to add DING for joiners and leavers #}
                {# "ivr_theme_name" : "Pexip theme (English_US) with entry tones", #}               
                "view" : "one_main_seven_pips",    {# pexCheck: The layout used in the merged VMR (shown on both SfB side and endpoint side) #}
                "automatic_participants" : 
                                [
                                {"remote_alias":"{{call_info.local_alias}}",
                                "local_alias":"VMR-{{s4buser}}", {# pexCheck: if you want the URI of the first endpoint, add {{call_info.remote_alias}} #}
                                "local_display_name":"VC", {# pexCheck: Name of the merged participant in the SfB meeting #}
                                "description":"ADP to SfB",
                                "protocol":"mssip",
                                "routing":"manual",
                                "role":"guest", {# pexCheck: change value from "guest" to "chair" if conference is set up with a PIN #}
                                "keep_conference_alive": "keep_conference_alive_never",
                                "system_location_name":"Location A" }
                                ]
                        }
        {% else %}
 
        {# If call_info.local_alias did not contain a GRUU, we proceed to check if normal service config would apply #}
 
           {% if service_config %}
                    "action" : "continue",
                    "result" : {{service_config|pex_to_json}}
 
            {# And if it does not, we will reject the call #}
 
            {% else %}
                    "action" : "reject",
                    "result" : {}
            {% endif %}
 
    {% endif %}
 
}

Policy #21

Schedule media playback to kick in based on weekday & time


Created this local policy for a customer that has a large all hands meeting every Monday morning that lasts for 1 hour and wants to allocate all port licenses and transcoding capacity to this meeting (at the expense of other meetings). The policy requires a media playback source called 'allhands_meeting_message' and a media playback list called 'allhands_meeting_message'. 

The script will:

Allow the all hands meeting with the service tag 'allowmeeting' at any time

Between 8-9 AM on Monday at the time of the all hands meeting re-direct participants attempting to join any other VMR's to a short media playback video called 'allhands_meeting_message' stating that 'a priority meeting is taking place, please reschedule your meeting..' or similar and then disconnects the call.

The policy will allow any participants to access any scheduled or static VMR at any other time.

 



{
{# Variables declared in lines 3-16 added to get in a weekday function #}

{% set offset = [0, 31, 59, 90, 120, 151, 181, 212, 243, 273, 304, 334] %}
{% set week   = ["Sunday", "Monday", "Tuesday", "Wednesday", "Thursday",  "Friday", "Saturday"] %}
{% set afterFeb = 1 %}
{% if now.month > 2 %}
    {% set afterFeb = 0 %}
{% endif %}
{% set aux = now.year - 1700 - afterFeb %}
{% set dayOfWeek0  = 5 %}
{% set dayOfWeek1 = dayOfWeek0 + (aux + afterFeb) * 365 %}
{% set dayOfWeek2 = dayOfWeek1 + aux / 4 - aux / 100 + (aux + 100) / 400 %}
{% set dayOfWeek3 = dayOfWeek2 + offset[now.month - 1] + (now.day - 1) %}
{% set dayOfWeek = dayOfWeek3 % 7 %}
{% set theDay = week[dayOfWeek | round(0) | int ] %}  
{% if service_config %}
    "action" : "continue",      
{% if service_config.service_tag.startswith("allowmeeting") %}
         "action" : "continue",
         "result" : {{service_config|pex_to_json}}
      
      {% elif theDay == "Monday" and now.hour == 8  %}
                    "action": "continue",
                    "result": {
                        "name": "allhands_meeting_message",
                        "service_type": "media_playback",
                        "service_tag": "allowmeeting",
                        "media_playlist_name": "allhands_meeting_message",

                  }
    {% else %}
         "action" : "continue",
         "result" : {{service_config|pex_to_json}}
    {% endif %}
  {% else %}
    "action" : "reject",
    "result" : {}
  {% endif %}
}
